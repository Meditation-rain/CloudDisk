1.项目概述

本项目要求参照悟空浏览器、夸克等主流产品的网盘功能，实现一个**简易的个人网盘客户端**。学员需通过该项目掌握移动端（Android 或 iOS）基础 UI 开发、本地存储、网络请求模拟及深层链接（DeepLink）等核心技能。

2.功能要求

A.首页

1. 网盘tab：个人信息，最近转存，最近浏览（主要实现 右图 红框内的功能）
   1. 最近浏览：根据文件打开顺序展现
   2. 最近转存：根据文件转存顺序展现
2. 文件tab：文件列表，至少包括以下文件类型：
   1. **文件夹：点击进入此文件夹的列表**
   2. **视频：点击打开拉起系统播放器**
   3. **文档（.txt）** **：打开文本阅读器**

B.文件管理

1. 分享：分享出一个链接，复制链接并打开app，可拉起分享的文件列表页，注意链接不包含直接的文件明文信息（无需跨设备转存）
2. 上传：选择设备本地文件，要求实现中使用到数据库
3. 文件删除、移动、重命名

C.txt文档阅读器：简易文本阅读器

要求：内容分页：左滑切下一页，右滑切上一页

D.拓展项

1. 架构：KMP + MVI 架构
2. 商业化：txt 文档页面切换时，插入广告（要求接入广告SDK，如穿山甲或优量汇 之一即可）
3. 增加服务端实现
4. 视频播放器：实现自定义的播放器

## 3. 技术与数据要求

- **UI 与架构**

- **UI 框架**：必须使用 **Compose** 实现（Android）或原生声明式 UI。
- **代码规范**：参照 GitHub 开源标准（Java/Kotlin/Swift 等），保持良好的命名与注释。
- **架构建议**：建议分层明确（UI 层、领域层、数据层分离）。

**数据与存储**

- **本地存储**：使用 SQLite (Room) 或其他移动端数据库，存储字段包含：`file_id`, `name`, `size`, `path`, `timestamp` 等。（如果服务端实现存储，这里就不强要求本地数据库）
- **文件操作**：可模拟网络请求，需要有数据解析，流程：json -> 数据库 -> 数据结构
- **异步处理**：网络模拟与数据库 IO 需在非 UI 线程执行，确保界面滑动无卡顿。
